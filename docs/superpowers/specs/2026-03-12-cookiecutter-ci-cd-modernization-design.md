# MLCookieCutter CI/CD & Tooling Modernization

## Summary

Port battle-tested CI/CD, pre-commit, semantic versioning, and linting configurations from xhai-llm-utils into the MLCookieCutter template. Add cookiecutter variables for DVC inclusion, test matrix, and default branch name.

## Approach

Direct port of xhai-llm-utils configs, adapted with Jinja2 conditionals for configurable features. Minimal deviation from proven setup.

## Cookiecutter Variables

Updated `cookiecutter.json`:

```json
{
  "directory_name": "DS_project",
  "project_name": "default_project",
  "project_description": "default_description",
  "python_version": ">=3.12",
  "target_python_version": "py312",
  "default_branch": "main",
  "include_dvc": false,
  "python_test_versions": "3.10, 3.11, 3.12"
}
```

- `default_branch` — used in CI/release workflows and semantic-release config. Default: `main`.
- `include_dvc` — toggles DVC deps (`dvc[s3]`, `dvclive`, `gto`) in pyproject.toml. Default: `false`.
- `python_test_versions` — comma-separated list, parsed in CI workflow to build a JSON matrix array.

## Build System & Versioning

### Build Backend

Switch from no build backend to hatchling + hatch-vcs:

```toml
[build-system]
requires = ["hatchling", "hatch-vcs"]
build-backend = "hatchling.build"
```

### Version Management

Hybrid approach: hatch-vcs for builds + semantic-release writes version to source files.

```toml
[tool.hatch.version]
source = "vcs"

[tool.hatch.version.raw-options]
version_scheme = "guess-next-dev"
local_scheme = "no-local-version"
```

Project version becomes dynamic:
```toml
dynamic = ["version"]
```

### Semantic Release Config

Full config matching xhai-llm-utils structure:

```toml
[tool.semantic_release]
version_toml = ["pyproject.toml:project.version"]
version_variables = ["src/__init__.py:__version__"]
branch = "{{cookiecutter.default_branch}}"
tag_format = "v{version}"
commit_message = "chore(release): {version}\n\nAutomatically generated by python-semantic-release"
build_command = """
    uv lock --upgrade-package {{cookiecutter.project_name}}
    git add uv.lock
    uv build
"""

[tool.semantic_release.changelog.default_templates]
changelog_file = "CHANGELOG.md"

[tool.semantic_release.remote]
type = "github"
token = { env = "GH_TOKEN" }

[tool.semantic_release.publish]
upload_to_vcs_release = true
```

Note on hybrid versioning: `dynamic = ["version"]` tells hatch-vcs to derive version from git tags at build time. Separately, `version_toml` tells semantic-release to write a static `version` field to pyproject.toml during releases. Both mechanisms coexist — hatch-vcs uses git tags as the source of truth for builds, while semantic-release keeps the toml and `__init__.py` in sync for code-level access.

### Version in Code

`src/__init__.py` will use `importlib.metadata` with a static fallback string that semantic-release can pattern-match and update (same approach as xhai-llm-utils):

```python
from importlib.metadata import PackageNotFoundError, version

try:
    __version__ = version("{{cookiecutter.project_name}}")
except PackageNotFoundError:
    __version__ = "0.0.0.dev0"  # updated by semantic-release
```

The `importlib.metadata` path is primary (used when package is installed). The fallback string `"0.0.0.dev0"` is what `semantic_release.version_variables` finds and replaces during releases. Both mechanisms coexist without conflict.

## Pre-commit Hooks

Complete replacement of `.pre-commit-config.yaml`:

1. **Local ruff hooks** (via `uvx`, `language: system`):
   - `ruff-check`: `uvx ruff check --fix --exit-non-zero-on-fix`, language: system, types: python, excludes `.ipynb`
   - `ruff-format`: `uvx ruff format`, language: system, types: python, excludes `.ipynb`

2. **uv-lock check**: `uv-pre-commit` repo — ensures `uv.lock` stays in sync

3. **General file safety** (`pre-commit-hooks` v5.0.0):
   - trailing-whitespace, end-of-file-fixer, check-yaml, check-toml
   - check-added-large-files (1000KB), check-merge-conflict
   - debug-statements, detect-private-key

4. **Conventional commit validation** (`conventional-pre-commit` v4.0.0):
   - Stage: `commit-msg`
   - Allowed types: feat, fix, docs, style, refactor, perf, test, build, ci, chore, revert

### Makefile Update

Add `--hook-type commit-msg` to `init_project`:

```makefile
init_project:
	git init
	uv sync
	uv run pre-commit install
	uv run pre-commit install --hook-type commit-msg
```

## GitHub Actions

### CI Workflow (`.github/workflows/ci.yml`)

Triggered on PRs to `{{cookiecutter.default_branch}}`.

**Jobs:**

1. **lint** (ubuntu-latest): Setup uv, install Python 3.12, `uv sync --frozen --group dev`, run ruff check + format check with GitHub output format.

2. **test** (ubuntu-latest, matrix): Python versions from `{{cookiecutter.python_test_versions}}` parsed into JSON array via Jinja2 (e.g., `"3.10, 3.11, 3.12"` becomes `["3.10", "3.11", "3.12"]`). `uv sync --frozen --group dev`, `uv run pytest --tb=short -v`. Fail-fast disabled. Deliberate divergence from xhai-llm-utils (which uses `--all-extras`): the cookiecutter template puts DVC deps in the dev dependency group, not in optional-dependencies, so `--group dev` is correct here.

3. **build** (depends on lint + test): Checkout with `fetch-depth: 0` (for hatch-vcs), `uv build`, twine check, upload dist artifacts.

Environment: `UV_CACHE_DIR: /tmp/.uv-cache`. All jobs use `astral-sh/setup-uv@v5` with `enable-cache: true` and `cache-dependency-glob: "uv.lock"`. Concurrency groups cancel in-progress runs.

### Release Workflow (`.github/workflows/release.yml`)

Triggered on push to `{{cookiecutter.default_branch}}`.

Permissions: `contents: write`, `id-token: write`. Concurrency: `release` (serialized).

**Steps:**
1. Checkout with `fetch-depth: 0` and `token: ${{ secrets.GITHUB_TOKEN }}` (required for pushing commits/tags back)
2. Configure git as `github-actions[bot]`
3. Setup uv + Python 3.12
4. Install deps + `python-semantic-release`
5. Get next version via `semantic-release version --print`
6. If new version: run `semantic-release version --no-push` (bumps version in pyproject.toml and src/__init__.py, creates commit but does not push). Then manually: update `uv.lock` via `uv lock --upgrade-package {{cookiecutter.project_name}}`, `git add uv.lock`, amend the semantic-release commit (`git commit --amend --no-edit`), force-tag `v$VERSION`, push to `{{cookiecutter.default_branch}}` with tags. Note: `build_command` in the semantic-release config is for when running `semantic-release publish` directly; the workflow handles the flow manually via `--no-push` + explicit steps (same pattern as xhai-llm-utils).
7. If released: build with `uv build`, create GitHub release via `gh release create "v$VERSION" dist/* --title "v$VERSION" --generate-notes`

## Dev Dependencies

Switch from `[tool.uv] dev-dependencies` to `[dependency-groups]` format:

```toml
[dependency-groups]
dev = [
    "pytest>=8.3.3",
    "ruff>=0.9",
    "pre-commit>=4.0.1",
    "build",
    "twine",
    "loguru>=0.7.2",
    "pyyaml>=6.0.2",
]

[tool.uv]
default-groups = ["dev"]
```

DVC dependencies (`dvc[s3]>=3.55.2`, `dvclive>=3.48.0`, `gto>=1.7.1`) conditionally included via Jinja2 when `cookiecutter.include_dvc == "true"`.

Removed: `interrogate`.
